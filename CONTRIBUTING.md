Contributing to TrueLazer

First of all, thank you for your interest in helping improve this project! We welcome contributions that help make this software better for everyone.

Because this project is Source-Available (and not Open-Source), there are specific rules regarding how you can contribute and what you can do with your modifications.
1. The Golden Rule: No External Distribution

By contributing to this project, you acknowledge that:

    You may not host your own version of this software on other websites, platforms, or public repositories.

    All development and distribution must stay within this GitHub repository.

    You may fork this repository only for the purpose of submitting Pull Requests back to this project.

2. How to Contribute

To ensure a smooth process, please follow these steps:

    Check Issues: See if the feature or bug you want to work on is already being discussed.

    Fork the Repo: Use the GitHub "Fork" button to create a copy of the code under your own profile.

    Develop & Test: Make your changes in a new branch.

    Submit a Pull Request (PR): Open a PR against our main branch. Provide a clear description of what you changed and why.

3. Contributor License Agreement (CLA)

By submitting a Pull Request, you agree to the following:

    You grant the project owner a non-exclusive, perpetual, irrevocable, and royalty-free license to include your contributions in the Software.

    Your contributions will be distributed under the project's existing LICENSE.

    You confirm that you are the original author of the code you are submitting and have the right to contribute it.

4. Code Standards

    Ensure your code is clean and well-commented.

    Follow the existing coding style used in the repository.

    If you are fixing a bug, please include a description of how to reproduce the bug.

Questions?

If you are unsure about the license or how to contribute, please open an Issue with the label question before you start coding. We are happy to help clarify!
